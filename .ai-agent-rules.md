# AI Agent Configuration
# Rules for preventing content analysis while maintaining file discovery

## Excluded Directories
These directories should NOT be read or analyzed by AI agents for security and performance reasons:

### Database Editor (Security Sensitive)
- `gycc-php/dbeditor/` - Contains database management tools and sensitive configurations
- Files to track for Supabase upload but NOT analyze:
  - Configuration files (*.php, *.inc.php)
  - Database schema files
  - SQL dump files

### Media Files (Content Analysis Not Required)
AI agents should only track file paths and metadata, not analyze content:

#### Video Files
- `*.mp4`, `*.avi`, `*.mov`, `*.wmv`, `*.flv`, `*.webm`, `*.mkv`, `*.m4v`
- Location mapping for Supabase storage buckets

#### Audio Files  
- `*.mp3`, `*.wav`, `*.ogg`, `*.flac`, `*.aac`
- Metadata extraction only (duration, format, size)

#### Image Files
- `*.jpg`, `*.jpeg`, `*.png`, `*.gif`, `*.bmp`, `*.tiff`, `*.svg`, `*.webp`, `*.ico`
- Basic metadata only (dimensions, format, size)

#### Document Files
- `*.pdf`, `*.doc`, `*.docx`, `*.ppt`, `*.pptx`, `*.xls`, `*.xlsx`
- File existence and path tracking only

## File Discovery Rules
Agents SHOULD discover and map:
1. File paths and directory structure
2. File metadata (size, modified date, type)
3. Supabase storage bucket mapping
4. Upload status tracking

Agents SHOULD NOT:
1. Read file contents from excluded directories
2. Analyze media file contents
3. Extract sensitive configuration data
4. Process database credentials or connection strings

## Supabase Upload Mapping
When uploading to Supabase, maintain this directory structure:

```
supabase-storage/
├── media/
│   ├── videos/
│   ├── images/
│   ├── audio/
│   └── documents/
├── dbeditor-backups/
│   ├── configs/
│   └── schemas/
└── public/
    └── assets/
```

## Implementation Notes
- Use file system APIs for discovery without content reading
- Implement content-type detection based on file extensions
- Maintain upload queue with file paths and metadata
- Log file operations without exposing sensitive content
